# AstrBot - 私聊历史记录插件

前情提要：此插件由astrbot完成 因此作者名字就是人格名字啦~
如果有bug可以让astrbot自己修改哦！！！

自动拉取指定 QQ 用户的私聊聊天记录，缓存到本地，并在 AI 对话时注入上下文，让 AI "记得"你之前聊过什么。

## 功能特性

- 自动缓存指定 QQ 用户的私聊记录
- AI 对话时自动注入最近的聊天历史作为上下文
- 智能过滤短消息（如"嗯""好的""哈哈"）
- 自动总结：当消息过多时调用 LLM 生成摘要，节省 Token
- 支持手动刷新：`/私聊历史` 命令随时触发

## 前置条件

- [AstrBot](https://github.com/Soulter/AstrBot) 已安装并运行
- [NapCat](https://github.com/NapNeko/NapCatQQ) 或其他 OneBot v11 协议端已配置并运行

## 安装方法

### 方式一：手动安装

1. 下载本仓库的 `private_chat_history` 文件夹
2. 将文件夹放入 AstrBot 的插件目录：
   ```
   <AstrBot根目录>/data/plugins/
   ```
3. 重启 AstrBot

### 方式二：通过 AstrBot 控制面板安装

1. 打开 AstrBot 控制面板（默认 `http://127.0.0.1:6185`）
2. 进入「插件」页面
3. 点击「安装插件」→「从本地安装」
4. 选择下载好的 `private_chat_history` 文件夹

## 配置说明

### 第一步：配置 NapCat 连接

在插件配置中填写你的 NapCat 信息：

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| HTTP 地址 | `http://127.0.0.1:3000` | NapCat 的 HTTP API 地址 |
| API Token | - | NapCat API 的授权 Token（查看 NapCat 配置获取） |

### 第二步：设置目标用户

在「目标用户 QQ 号」中填写需要拉取私聊记录的 QQ 号。

- 支持多个用户，用逗号或换行分隔
- 示例：`123456789,987654321`

### 第三步：调整参数（可选）

#### 基础设置

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| 每次拉取消息数量 | `20` | 每次从 API 拉取的消息条数，建议 20-50 |
| 最大缓存条数 | `60` | 本地缓存的最大消息条数，超出自动截断旧消息 |
| 缓存刷新间隔（秒） | `300` | 两次刷新的最小间隔（防止频繁请求） |

#### 上下文注入

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| 自动注入私聊历史 | `开启` | 每个会话首次对话自动注入；关闭则只响应命令 |
| 最大注入消息数 | `15` | 注入到 AI 上下文的最大消息条数 |
| 最小注入消息数 | `5` | 最少需要注入的消息数，不足时放宽过滤条件 |
| 最小消息长度 | `5` | 过滤短消息的阈值（如"嗯""好的"不会注入） |

#### 自动总结

当注入文本超过字数阈值时，自动调用 LLM 生成摘要，后续使用缓存的摘要。

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| 消息 10-20 条时总结阈值 | `400` | 缓存 10-20 条消息，文本超过 400 字触发总结 |
| 消息 20-30 条时总结阈值 | `500` | 缓存 20-30 条消息，文本超过 500 字触发总结 |
| 消息 30-40 条时总结阈值 | `600` | 缓存 30-40 条消息，文本超过 600 字触发总结 |
| 消息 40-50 条时总结阈值 | `700` | 缓存 40-50 条消息，文本超过 700 字触发总结 |
| 总结使用的模型 | `（空）` | 留空使用当前聊天模型，可指定 Provider ID |

## 使用方法

### 自动模式（推荐）

1. 启动 AstrBot 和 NapCat
2. 在控制面板开启「自动注入私聊历史」
3. 向 AI 发送第一条消息时，会自动注入最近的私聊记录

### 手动模式

发送 `/私聊历史` 命令：
- 立即拉取最新的私聊记录
- 生成总结并注入当前会话

### 工作流程

```
用户发消息 → AstrBot 处理 → 插件拦截 → 读取缓存 → 格式化 → 注入 AI 上下文 → AI 回复
```

**注意：每个会话只注入一次**（避免重复浪费 Token）。如需刷新：
- 发送 `/私聊历史` 命令
- 或切换到新会话

## Token 消耗参考

| 场景 | Token 消耗 |
|------|-----------|
| 自动注入（无总结） | 约 1000-3000 |
| 自动注入（有总结） | 约 300-500 |
| 手动 `/私聊历史` 生成总结 | 约 1000-2000（一次性） |
| 系统提示词 + 工具定义 | 约 6000-10000 |
| 每轮对话 | 约 2000-5000 |

## 常见问题

### AI 不记得之前聊的内容？

检查以下几点：
1. 「自动注入私聊历史」是否开启
2. 「目标用户 QQ 号」是否正确填写
3. NapCat 是否正常运行
4. 查看 AstrBot 日志中的 `[private_chat_history]` 信息

### 总结生成失败？

检查「总结使用的模型」配置：
- 留空：使用当前聊天模型
- 如果当前模型不可用，指定其他可用的 Provider ID

### 缓存文件太大？

「最大缓存条数」默认 60 条，超出自动截断旧消息。可调小（如 40）。

### 每次只注入一次，切换话题后怎么办？

这是设计如此——避免重复注入浪费 Token。如需刷新：
- 发送 `/私聊历史`
- 或在 AstrBot 中切换到新会话
